## TL;DR

EdgeProbe is observability tooling for on-device AI on mobile. Original pitch was "Langfuse for edge" — broad cross-runtime cloud SaaS. After CEO review, the plan is sharpened to a **hybrid B+C strategy**: content-led discovery in Year 1 (monthly benchmark posts + voice AI hardware postmortems) feeding a productized voice-agent observability wedge in Year 2 (iOS first, llama.cpp + Whisper.cpp only, cloud dashboard with public-by-default shareable URLs driven by a CI GitHub Action).

The plan's biggest risks are: (1) Apple sherlocking iOS-side AI profiling, (2) market size for on-device AI dev tools is small (~2-5K teams globally), (3) solo-founder bandwidth across content + product + ops. Mitigations: cross-runtime + cross-platform vendor-neutrality, content-as-validation-loop before product investment, kill-criteria framework with named accountability partner.

## Data Model — Three-tier session/conversation/turn (OTel-compatible)

**AppSession is NOT a span — it's an attribute propagated on every Turn span.** This is a critical correction from naive nested-spans modeling: AppSession can last hours/days, which breaks BatchSpanProcessor (root spans never end → never flushed). Industry pattern (OTel, Datadog, Honeycomb for session tracking): session.id flows as an attribute, not as an enclosing span.

Conversation IS a span (short-lived, seconds to minutes), with Turn spans as children.

```
  AppSession  ──> just a session.id attr, NOT a span
  │             propagated onto every Conversation and Turn
  │
  Conversation (root span — kind=conversation, starts when user opens agent, ends when they leave)
    attrs: conversation.id (uuid v7), session.id (attr propagated from AppSession),
           conversation.mode (voice|text|mixed), app.version, device.model, os.version,
           net.connection_type, edgeprobe.user_id (hashed, opt-in)
    │
    ├── Turn (child span — kind=llm)
    │     gen_ai.request.model, gen_ai.request.temperature,
    │     gen_ai.usage.input_tokens, gen_ai.usage.output_tokens,
    │     gen_ai.response.finish_reasons, gen_ai.operation.name=chat,
    │     session.id (propagated)
    │     + device perf: edgeprobe.tokens_per_sec, edgeprobe.ttft_ms,
    │                    edgeprobe.memory_peak_mb, edgeprobe.thermal_state
    │
    └── Turn (child span — kind=stt/tts/embedding/…)
          gen_ai.request.model (whisper-small, etc.),
          session.id (propagated)
          + edgeprobe.audio_duration_ms, edgeprobe.wer (if ground truth),
            edgeprobe.rtf (realtime factor)
```

AppSession's lifecycle is managed by the SDK (generate UUID on app launch, persist across backgrounding, rotate on user sign-out) and exposed as `EdgeProbe.currentSessionId`. All spans carry it as an attribute. Dashboard groups by session.id to render "session" views without needing an actual session span.

Postgres schema (ingest-side normalization):

```
  conversations:   id (= conversation.id from span), org_id, session_id,
                   device_model, os_version, app_version, mode, started_at, ended_at
  turns:           id (= span id), conversation_id, session_id, kind, model,
                   input_tokens, output_tokens, ttft_ms, tokens_per_sec,
                   memory_peak_mb, thermal_state, duration_ms, finish_reason,
                   error, raw_span_jsonb (full OTel span for replay)
  share_reports:   id, turn_ids[], unguessable_token, expires_at, org_id_nullable
```

No `app_sessions` table — `session_id` is a denormalized column on conversations + turns for query grouping. Session "records" are views: `SELECT session_id, min(started_at), max(ended_at), count(*) FROM conversations GROUP BY session_id`.

Raw OTel spans archived in S3 (`spans/{org_id}/{yyyy-mm-dd}/{span_id}.json`) for reprocessing. Normalized tables power the dashboard queries; raw spans are the source of truth.

OTel mapping notes:
- `session.id` + `conversation.id` are custom attributes (OTel GenAI spec hasn't landed conversation.id as stable yet — pin to draft spec and document migration).
- `edgeprobe.*` namespace for device-side metrics OTel GenAI doesn't cover yet.
- When OTel GenAI semconv reaches stable, re-emit under stable names; keep legacy names for one minor version.

## Test Strategy (Y2 P0 — required, not optional)

Observability SDKs ship on thousands of devices. Bugs are public. Test plan is part of the product spec.

### SDK (iOS, Swift)
- **Unit tests (XCTest):** every public API. `EdgeProbe.start`, `.trace`, exporter, queue, shim.
- **Integration tests (XCUITest) in a host app:** real-device-like runs against both mocked llama.cpp and real llama.cpp binaries. Verify spans flow end-to-end through OTLP export.
- **Contract tests:** snapshot exact OTLP wire output for representative traces. Pin these — any unintentional schema change fails CI.
- **Critical regression tests (from eng review test diagram):**
  - Public share URLs never include prompt/completion content (assert on response shape)
  - Auth'd trace endpoint blocks cross-org access
  - Per-call `includeContent: true` does NOT leak to public URL
  - Drop-oldest on buffer overflow with counter emitted
  - Main thread never blocked on network (thread-sanitizer + XCTestExpectation timing assert)
  - Idempotent `start()` (called twice = no double init)

### Backend (Railway/Fly + Postgres)
- **API contract tests:** every endpoint tested for happy path + auth failure + rate limit + malformed input.
- **Postgres migration tests:** every migration runs forward + backward on an ephemeral DB in CI.
- **E2E tests (Playwright):** developer-integrates-SDK flow, share-URL flow, auth'd-dashboard flow.

### CI Matrix
- SDK built against llama.cpp tagged versions: current, current-1 minor, current-2 minor.
- Swift toolchain: current Xcode + current-1 Xcode.
- iOS simulator: iOS N, N-1.
- Green CI required before merge — no skip flags.

### XCFramework build automation (critical — one-time investment)
- Dedicated GitHub Actions workflow in the SDK repo: on detection of a new llama.cpp upstream tag, auto-build XCFrameworks for the full matrix (Xcode × iOS × llama.cpp versions) and upload to GitHub Releases.
- Uses `macos-latest` runners + `ccache` for compile speed.
- Solo founder's job: review the PR, run integration tests, merge. Not "build 12 XCFrameworks by hand."
- Without this, llama.cpp version bumps eat 20-30% of ongoing eng time (per outside voice). This workflow is an ~1-week one-time build that pays for itself within two upstream releases.
- Same pattern for Whisper.cpp.

### Benchmark harness (Y1, OSS)
- Reproducibility: deterministic seeds, fixed input prompts, hash output tokens to catch drift.
- CI runs the harness against a matrix of llama.cpp versions and posts a diff comment on PRs.
- This harness is ALSO the fixture source for SDK integration tests.

### Coverage target
- SDK: ≥90% line coverage, 100% of public API tested.
- Backend: 100% endpoints + 100% auth boundaries.
- The 6 CRITICAL items in the test diagram are blocking — no ship without them.

## Architecture Decisions (Day 1, sticky)

- License: **MIT** (SDK + GitHub Action + harness)
- Telemetry schema: **OpenTelemetry GenAI semantic conventions** (gen_ai.* attributes), emitted via the official `opentelemetry-swift` SDK. EdgeProbe's contribution is a thin enrichment layer that maps llama.cpp / Whisper.cpp events to gen_ai.* spans and ships via OTLP. Reuses BatchSpanProcessor for offline queuing + retry + backoff — no custom ring buffer. Gives "lateral integration trivial" with Langfuse/Datadog/OTel collectors on day 1.
- SDK stack: **Swift native**; llama.cpp + Whisper.cpp consumed as precompiled XCFrameworks via `binaryTarget()` in Package.swift (avoids the known `unsafeFlags()` SPM breakage from llama.cpp's official Package.swift — ggml-org/llama.cpp#10371). Reference implementation: Stanford SpeziLLM. No OTHER third-party deps in the core instrumentation path (binary size + maintenance).
- Backend stack: **Railway or Fly.io with co-located Postgres + S3-compatible storage** (revised from Vercel+Supabase per eng review — single platform, co-located DB, ~$30-60/mo vs ~$150+/mo)
- Hosting: **US-only** (us-east-1 region, multi-region deferred)
- Default privacy posture: **content NEVER uploaded by default.** What travels: span structure, timing, token counts, device model, OS version, model name, thermal state, finish reasons, error codes. What does NOT travel unless explicitly opted in per-call: prompt text, completion text, audio bytes, transcripts, user identifiers, tool call arguments/results. No regex-based "PII redaction" — regex is famously broken (emails in base64, names in non-Latin scripts, PII in freeform audio). Opt-in is a single explicit API: `EdgeProbe.trace(.llm, includeContent: true) { ... }`. Per-call. Never a global flag.
- Report URL pattern: **two distinct URL surfaces, enforced at the data plane:**
  - `/r/{unguessable_token}` — public, no auth needed. Shows span structure + timing + device + model names + perf metrics + error codes. NEVER renders prompt/completion/audio content, even if content was opted-in on the backend. 30-day expiry on free tier.
  - `/app/trace/{id}` — auth'd, org members only. Full content access if uploaded. Used for internal debugging.
  The two URL patterns read from different API endpoints that apply different field projections at the DB layer — not a UI toggle.
- Public URL side-channel threat model:
  - Risk: even without content, token counts + latency + prompt length + finish-reason patterns can fingerprint prompts (e.g., "1247 input tokens + 89 output tokens on medical-triage model" reveals category of query).
  - Mitigations for `/r/{token}`:
    - Token counts bucketed (round to nearest 10% for small N, log-scale for large N)
    - Timing data reported as session-level p50/p99 aggregates, not per-turn raw values
    - `EdgeProbe.trace(.llm, sensitive: true) { ... }` flag strips a turn entirely from public views (perf metrics only at session-level aggregate)
    - Residual side-channel risk documented in privacy policy so customers can make informed exposure decisions
  - The `/app/trace/{id}` auth'd surface retains raw values for internal debugging.
- Customer contracts: **assignable, no exclusivity clauses**
- Ingest auth: **two-key model.** Each org gets (1) a `epk_pub_...` public ingest key that ships in their iOS bundle and can only write spans, and (2) a `epk_priv_...` private dashboard key that can read. Public key leaks are expected and survivable — defended with per-org rate limits (spans/sec, MB/day), content-hash dedup to kill replay floods, and one-click public-key rotation that invalidates the old key (customer ships a new app build). Private key never leaves server-side dashboards or CI secrets.
- Brand: **EdgeProbe** (already chosen, locked)

## Design Decisions (Day 1, sticky)

Design is not a pre-launch polish pass. The public share URL `/r/{token}` IS the viral top-of-funnel — it carries the brand to every Slack channel and Twitter unfurl it lands in. These decisions are committed Day 1 so month-13 code implements them instead of inventing them.

### Brand voice

Engineer-to-engineer. Numbers over adjectives. Zero hype words. No "powerful," "seamless," "effortless," "magical," "revolutionary," "cutting-edge." Headlines state the measurement, not the feeling: "Whisper-tiny on iPhone 15 Pro — 34% slower than baseline" beats "Unlock on-device AI insights." Copy inspiration: Linear changelog + Fly.io blog + Vercel CLI output.

### Visual system

- **Typography:** IBM Plex Mono (data, durations, device IDs, code) + Inter (prose, headlines, UI labels). Both free, both Google Fonts, both well-rendered across browsers. No default system font stack.
- **Color system (CSS custom properties, dark-first):**
  - Background: `--bg: #0a0a0b` (near-black, NOT pure black)
  - Surface: `--surface: #131316`
  - Border: `--border: #26262b`
  - Text primary: `--fg: #f5f5f7`
  - Text muted: `--fg-muted: #9a9aa1`
  - Accent (timing bars, links): `--accent: #4cc9f0` (restrained cyan; not purple, not indigo, not brand-gradient soup)
  - Signal regression: `--signal-bad: #ef4444` (red-500, used ONLY for regressions/errors)
  - Signal ok: `--signal-ok: #22c55e` (green-500, used ONLY for passing thresholds)
  - No yellow "warning" tier on public URLs — binary pass/fail reads faster to a cold viewer
- **Spacing:** 8px base grid. Scale: 4, 8, 12, 16, 24, 32, 48, 64.
- **Radii:** 6px default, 2px on data density elements (span bars, inline chips). No bubbly large radii.
- **Shadows:** none by default. A premium interface reads premium without decorative shadows. Use borders + surface elevation.
- **Max content width:** 960px on `/r/{token}`. Wider feels dashboardy. Narrower feels blog.
- **Mode:** dark-default. `prefers-color-scheme` respected; light mode is a secondary track, shipped by month 18, not month 13.

### Accessibility baseline

- **Contrast:** all text-on-background pairs pass WCAG 2.2 AA (4.5:1 for body, 3:1 for large text). `#f5f5f7` on `#0a0a0b` = 18.6:1. `#9a9aa1` on `#0a0a0b` = 7.2:1. Verify with automated axe run in CI.
- **Keyboard nav:** tab order is semantic (skip-to-content link, then utility bar, then H1, then metric strip, then timeline, then footer). Focus ring is a 2px `--accent` outline, never `outline: none`.
- **Screen reader:** Gantt timeline has a hidden `<table>` fallback with the same per-turn data (screen readers skip the visual Gantt). Lock icons on redacted content slots have `aria-label="Content not shared for this turn"`.
- **Touch targets:** 44×44px minimum for any tap target on mobile.
- **Reduced motion:** `prefers-reduced-motion: reduce` disables the one-time entrance animation on metric numbers; all values render immediately.
- **Color is not the only signal:** regressions get a red bar AND a `▲ 34%` delta arrow AND explicit text ("34% slower than baseline"). Never color-alone.

### Motion

Two motions, both earn their pixels:
1. **Metric count-up** on first paint (300ms, ease-out, disabled under `prefers-reduced-motion`). Signals "live measurement, just rendered" rather than a static image.
2. **Span bar fill** on the Gantt (staggered 40ms between bars, 600ms total). Helps the eye absorb relative durations instead of seeing a wall of rectangles at once.

No hover bounces, no parallax, no scroll-linked decorative effects. If it's not communicating hierarchy or state, it's cut.

### Architecture decisions recorded (Year 2 P0)

1. **Backend stack:** Railway or Fly.io with co-located managed Postgres. Kills Vercel+Supabase cold-start + geo-split tail latency pattern for OTLP ingest.
2. **SDK distribution:** XCFramework via `binaryTarget()` for llama.cpp + Whisper.cpp. Swift Package Manager's `unsafeFlags` breaks downstream integration (ggml-org/llama.cpp#10371). Reference: Stanford SpeziLLM uses the same pattern.
3. **Telemetry:** opentelemetry-swift SDK + BatchSpanProcessor (off main thread). Semantic conventions follow `gen_ai.*` GenAI draft; flagged as DRAFT in plan so we don't ship frozen schema and then chase upstream.
4. **Span model:** Turn span = root. `session.id` propagated as attribute across turn spans (NOT an enclosing AppSession span). Reason: long-lived root spans leak memory and break BatchSpanProcessor assumptions. No `app_sessions` table needed for P0.
5. **Ingest:** synchronous Postgres writes for P0. Redis + worker queue deferred until P50 ingest latency > 200ms OR PMF. Premature async = premature complexity.
6. **Auth:** two-key model. `epk_pub_` public ingest key (shippable in app binary, rate-limited, rotatable). `epk_priv_` private dashboard key (server-side only).
7. **URL surfaces:** two distinct surfaces enforced at the data plane. `/r/{token}` public share = aggregates + timings only, NEVER prompt/completion text. `/app/trace/{id}` auth'd = full content if org members opted in per-call.
8. **PII model:** content NEVER uploaded by default. Per-call opt-in via `EdgeProbe.trace(.llm, includeContent: true) { ... }`. Opting in does NOT escalate public visibility.
9. **SDK behavior:** bounded ring buffer (host-configurable size), drop-oldest on overflow, dropped-span counter emitted as metric on reconnect. OSLog for errors. Main thread never blocked on network or serialization.
10. **Binary size:** <500KB wrapper + <2MB total with XCFramework. GitHub Actions builds XCFrameworks on llama.cpp release tags automatically — founder does not hand-build on every upstream release.
11. **CI matrix:** llama.cpp current + current-1 + current-2 minor; Xcode current + current-1; iOS simulator N + N-1. No skip flags allowed.
12. **Side-channel threat model:** token bucketing + session-level aggregates only on public URLs + `sensitive: true` flag path for operators to suppress publicly. Documented so we don't accidentally leak user behavior via timing/frequency channels.

### Design decisions recorded (Year 2 P0)

1. **Brand voice:** engineer-to-engineer, numbers over adjectives, banned hype vocabulary enumerated in `DESIGN.md`.
2. **Typography:** IBM Plex Mono (data, durations, code) + Inter (prose, UI). No default system font stack.
3. **Color system:** dark-first with CSS custom properties. `--bg: #0a0a0b`, `--accent: #4cc9f0`, `--signal-bad: #ef4444`, `--signal-ok: #22c55e`. No yellow warning tier on public URLs (binary pass/fail).
4. **Spacing:** 8px base grid, 4/8/12/16/24/32/48/64 scale.
5. **Radii:** 6px default, 2px on data-dense elements. No bubbly radii.
6. **Shadows:** none by default. Elevation via borders + surface color.
7. **Max content width:** 960px on `/r/{token}`, 1280px on `/app`.
8. **IA anchor on `/r/{token}`:** Verdict H1 at the top (Inter 48/56), single-column vertical trace document. NOT a dashboard, NOT a card grid.
9. **IA on `/app`:** list-row pattern, top nav horizontal (Projects · Sessions · Settings), no left sidebar at P0.
10. **OG image scope:** promoted from P2 to P0. Viral loop depends on the Slack/Twitter unfurl looking deliberate, not auto-generated.
11. **PR comment template:** P0, alongside the GitHub Action. Plain GFM tables, `<details>` collapse for per-turn diff, one link ("View full trace →") above the fold.
12. **Interaction states:** every surface specifies loading, empty, error, expired, partial. Captured in the Design Decisions coverage table.
13. **Responsive:** mobile is the primary target for `/r/{token}` (Slack/Twitter webviews). 2×2 metric grid on mobile, horizontal-scroll Gantt with affordance.
14. **Accessibility:** WCAG 2.2 AA baseline, 18.6:1 primary contrast, axe in CI blocking new violations, hidden `<table>` fallback under Gantt for screen readers.
15. **Motion:** two motions only (metric count-up, span bar staggered fill). Both disabled under `prefers-reduced-motion`.
16. **Forbidden patterns:** enumerated in `DESIGN.md` — no SaaS card grid as main layout, no purple gradients, no 3-column feature grid, no centered-everything, no emoji beyond 5 allowed glyphs, no shadows, no yellow warning tier on public surfaces.
