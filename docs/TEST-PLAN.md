## Key Interactions to Verify

### iOS SDK
- 3-line install (`import EdgeProbe; EdgeProbe.start(apiKey:); EdgeProbe.trace(.llm) { ... }`) from scratch in a fresh host app works
- `EdgeProbe.trace(.llm) { ... }` captures a span visible in dashboard within 60s
- `EdgeProbe.trace(.llm, includeContent: true)` uploads content; default (false) does NOT
- Nested `.trace()` calls produce correctly nested OTel spans
- `start()` called twice is idempotent
- OSLog surfaces SDK errors + dropped-span counts

### Dashboard

- Authenticated dashboard hides content from non-members

## Edge Cases

- SDK: backend unreachable → spans queue into ring buffer
- SDK: buffer overflow → drop-oldest, counter incremented, counter ships as metric on reconnect
- SDK: app backgrounded mid-export → resume on foreground
- SDK: app killed mid-export → buffer persists across launches
- SDK: main thread never blocked on network or serialization (ThreadSanitizer + timing assertions)
- Ingest: invalid OTLP → 400 with clear error
- Ingest: per-org rate limit exceeded → 429 with `Retry-After` header
- Ingest: payload > size limit → 413
- Ingest: replay flood (same content hash) → dedup silently
- Ingest: public key rotation → old key 401s, new key 200s
- Dashboard: expired share URL → 404 (not 500)
- Dashboard: cross-org trace-id scan → 403 (not 404 — must NOT leak existence)

## Critical Paths (★CRITICAL — regression rule; no ship without test)

1. **Public share URL PII guarantee:** prompt/completion text NEVER in `/r/{token}` response even if content was uploaded for that span
2. **Cross-org auth isolation:** `/app/trace/{id}` blocks access from other orgs (not 404 — 403, so we don't confirm existence)
3. **Per-call opt-in does not escalate visibility:** opting in to content upload does NOT make that span visible on public URLs
4. **Main thread never blocked by SDK:** network + serialization on background DispatchQueue
5. **Drop-oldest on buffer overflow:** bounded memory during backend outage; dropped-span counter emitted as metric on reconnect
6. **Idempotent SDK initialization:** `start()` called twice does not double-init, does not duplicate exports
